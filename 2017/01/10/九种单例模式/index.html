java 九种单例模式

发表于 2017-01-10   |     |   阅读次数

单例模式大家都很常见了，我这篇文章是看了别人的文章后自己学习后写出来的，相当于一个笔记吧。

饿汉式

  • 优点：线程安全

  • 缺点：由于类加载机制，无论类是否被使用，类第一次加载进入内存就回实例化

饿汉式 – 静态代码块

  • 优点:线程安全

  • 缺点：类装载时创建实例，若实例一直未被使用时，会浪费资源

懒汉式

  • 优点：避免了第一次加载这个类进入内存未使用占用内存

  • 缺点：线程不安全,当多线程情况下 getInstance时可能会生成多个实例

懒汉式 – 方法同步

  • 优点：线程安全

  • 缺点：同步方法，实际上只需要处理 new Singleton 的同步问题即可，效率太低了

懒汉式 – 代码块同步

  • 优点：效率高于 方法同步 的懒汉式

  • 缺点：判断 singleton 是否为空的时候，如果此时多线程访问了，那么也存在异步问题，实例化了多个singleton

懒汉式 – 错误的 double check

  • 优点：效率高

  • 缺点：线程不安全，依然存在创建多个实例的可能。因为每个线程都有自己的一份拷贝，并不能保证实例化后将INSTANCE的引用拷回主内存，不能保证对其它线程立即可见，所以仍然有可能造成多个实例被创建，具体原因搜索 volatile 关键字

懒汉式 – 正确的 doucle check

  • 优点：效率高，解决了线程不安全的问题

枚举

  • 优点: 无线程同步问题，且能防止通过反射创建新的对象

  • 缺点: 使用的是枚举，而非类。同时单一实例的访问点也不是一般单例模式的静态方法

静态内部类

  • 优点： 静态内部类不存在类加载的时候就初始化，当加载内部类的时候，java 虚拟机初始化其成员变量并且保证线程安全，而且没有锁不会造成性能影响

总结

以上就是我的笔记，总的来说一般我常用静态内部类的模式，最终的模式选择取决于大家。
